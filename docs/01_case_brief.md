# Case Brief

## Project Title

**IT GRC Risk & Control Assessment Case Study**

## Case Scenario

This project is based on a simulated internal academic services system used by an education-focused organization.

The system supports core academic and administrative services, including:

- student profile management,
- course registration,
- payment status visibility,
- academic document request,
- reporting dashboard,
- administrative user access,
- system change handling,
- incident support,
- backup and recovery.

The scenario is intentionally simulated because real IT GRC data, risk registers, access reviews, evidence files, audit findings, and remediation plans are normally confidential.

---

## Business Background

Academic service systems often process sensitive data and support time-critical operations.

A disruption, unauthorized access, weak change control, incomplete backup testing, or poor incident escalation can create operational, compliance, and reputational risk.

Management therefore needs a structured IT GRC view that connects:

- important assets,
- key IT risks,
- current controls,
- control effectiveness,
- evidence availability,
- control gaps,
- remediation plans,
- and risk reporting.

---

## Problem Statement

The simulated organization does not yet have a structured IT GRC assessment package that clearly documents:

1. which IT assets are in scope,
2. what risks affect those assets,
3. which controls exist,
4. whether the controls are properly designed and implemented,
5. whether evidence is available,
6. which control gaps require remediation,
7. who owns each corrective action,
8. and how management can monitor risk and remediation status.

Without this structure, risk ownership, evidence collection, remediation priority, and management reporting may become inconsistent.

---

## Project Objective

The objective of this project is to create an IT GRC assessment package that demonstrates how risks, controls, evidence, and remediation actions can be documented and monitored in a structured way.

The project will produce:

- asset inventory,
- risk register,
- control matrix,
- control assessment results,
- gap analysis,
- remediation plan / POA&M,
- evidence tracker,
- SQL monitoring queries,
- and management summary.

---

## Assessment Scope

### In Scope

The project covers the following IT GRC areas:

| Area | Scope Description |
|---|---|
| Access Management | User access, admin roles, periodic access review, least privilege |
| Data Privacy | Protection of personal and sensitive academic data |
| Backup & Recovery | Backup availability, restoration testing, recovery readiness |
| Change Management | Approval, documentation, UAT evidence, and release control |
| Incident Handling | Incident triage, severity, escalation, and response documentation |
| Logging & Monitoring | Admin activity review, log availability, suspicious activity detection |
| Vendor Dependency | Third-party/system dependency and SLA review |
| User Awareness | Basic security and data handling awareness |
| Data Quality | Reporting reliability, completeness, and validation |
| Business Continuity | Continuity planning for critical academic services |

### Out of Scope

This project does not include:

- penetration testing,
- vulnerability exploitation,
- production system audit,
- legal compliance opinion,
- real employee or student data,
- official ISO 27001 audit,
- official COBIT assessment,
- or certification readiness engagement.

---

## Key Stakeholders

| Stakeholder | Interest / Decision Need |
|---|---|
| Management | Understand top IT risks, remediation priority, and overall risk posture |
| IT Manager | Track IT risk ownership, control gaps, and remediation progress |
| System Owner | Ensure changes, access, and service continuity are managed properly |
| IT Operations | Maintain backup, logging, incident, and system support readiness |
| Data Owner | Protect personal and sensitive academic data |
| Internal Audit / GRC Support | Review evidence, control gaps, and remediation follow-up |

---

## Key Business Questions

This project is designed to answer:

1. What are the key IT risks affecting the internal academic services system?
2. Which assets are most sensitive or critical?
3. Which existing controls reduce those risks?
4. Which controls are missing, partial, or weak?
5. What evidence is required to support audit-readiness?
6. Which corrective actions should be prioritized?
7. Who owns each risk and remediation action?
8. How can management monitor risk, control gaps, and remediation status?

---

## Expected Deliverables

| Deliverable | Purpose |
|---|---|
| Asset Inventory | Defines assessment scope and business criticality |
| Risk Register | Documents risk statements, scoring, owners, and responses |
| Control Matrix | Maps risks to control objectives, activities, evidence, and source frameworks |
| Control Assessment Results | Evaluates design, implementation, operating effectiveness, evidence, and findings |
| Gap Analysis | Explains current condition, target condition, gap, risk, and priority |
| Remediation Plan / POA&M | Tracks corrective actions, owners, dates, status, evidence, and closure criteria |
| Evidence Tracker | Supports audit-readiness and evidence monitoring |
| SQL Monitoring Queries | Enables reporting on high risks, weak controls, open actions, and missing evidence |
| Management Summary | Provides executive-level overview and recommendations |

---

## Success Criteria

The project is considered successful if it produces a clear, recruiter-readable IT GRC portfolio package that shows:

- structured risk identification,
- source-based control mapping,
- practical control assessment,
- evidence-aware audit thinking,
- remediation planning,
- and management-level communication.

---

## Assumption

All data in this project is synthetic and created for learning and portfolio demonstration.

The project is designed to show how IT GRC artifacts can be structured, not to claim that a real audit or official certification assessment was performed.
